# Project Instructions

## Project Overview
This is a creative personal website project using vanilla HTML, JavaScript, and CSS. UI changes are iterative — expect multiple rounds of visual refinement. Always use the browser preview to verify visual changes before reporting completion.

## Git & Commits
- **No** `Co-Authored-By` lines or AI attribution in commits
- Write commit messages in **third person present** (e.g. "Adds blursed hub page", "Removes navbar from index")
- Keep messages short — one line unless a second line genuinely helps
- Use concise, lowercase commit messages without conventional commit prefixes
- Do not auto-attribute Claude in commits unless asked

## Development Workflow
- When making CSS/styling changes, always append a cache-busting query parameter (e.g., `?v=timestamp`) to stylesheet and asset URLs, or remind the user to hard-refresh. Browser caching frequently causes visual changes to not appear.

## UI Changes
- Use the **frontend-design** skill when making visual or layout changes
